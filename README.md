# 水準ボイス野帳

水準測量の現場で、スマートフォン・タブレット・パソコンから観測値を入力し、音声入力を併用して野帳を作成できる無料の静的Webアプリです。AI APIや有料APIは使用していません。

制作：株式会社アイズ測量 / Ezアイズ Survey Tools  
公式ホームページ：https://ez-survey.sakura.ne.jp/

## 主な機能

- 往路・復路を切り替える2枚の水準野帳（各200行、No.欄から任意行の追加・削除）
- 器械高方式による往路・復路ごとの高低差、行別の往復差、往復閉合差
- 手入力の既知標高を保護した仮標高の連続自動計算
- Web Speech APIによる日本語音声入力
- Enterキーでは同じ列の真下へ移動
- 認識した値の音声復唱
- 0.5～1.5倍で変更できる復唱速度
- 数値が小数点込み5文字になった時点で音声認識を確定
- iPhoneで音声入力中に画面キーボードを再表示しないセル移動
- 音声認識・復唱中は全セルのiPhoneキーボードを抑止し、タップしたセルへ音声入力先だけを変更
- iPhoneでタップとスワイプを判別し、スクロール中はセル選択を変更しない
- 横長ボタンで継続する音声入力モードへ切り替え、モード中は全セルのキーボードを抑止
- FSへの音声入力後は、次の行の点名へ移動
- 野帳表のセルだけを拡大・縮小する2本指ピンチ操作
- 往路・復路切替時の点名相互反転（空欄の位置を保持）
- 往路・復路の行数、行追加、行削除を常に同期
- BS・FSの10m以上の入力を拒否
- BS・FSの音声値を小数3桁で検査し、複数候補が食い違う場合は入力せず再認識
- 設定した音声の読みを点名へ優先変換
- 測量用語の優先辞書と、入力内容に応じた軽量な測点名候補
- 現在行より上の点名種類を使った、最大3件の増番候補
- シート名の横に配置した青い矢印で、往路・復路ごとに50操作まで「元に戻す／やり直す」
- 通常は非表示で、必要なときだけ表示できる距離列
- localStorageへの上書き保存・自動復元・シート別消去
- UTF-8 BOM付きCSV出力
- 作業規程に基づく1～4級の選択と、距離に応じた許容値・「合格／要確認」の自動表示
- PWA（ホーム画面追加・オフライン起動）
- スマートフォン、タブレット、パソコン対応

## 使用方法

1. シート名の横にある「復路へ」または「往路へ」で入力するシートを切り替えます。
2. 最初の既知標高を「既知/仮標高」欄へ入力します。空欄の場合は0mから計算します。
3. 最初の点で後視BSを入力し、次の点から前視FSを入力します。FSとBSが同じ行にある場合は、FSで標高を求めた後、その行のBSで新しい器械高へ切り替えます。
4. 閉合差と判定を上部の集計欄で確認します。

No.の数字を押すと、その行の下への行追加または選択行の削除ができます。往路と復路は常に同じ行数になり、追加・削除は両方のシートへ同時に反映されます。
シート名の横にある1個の切替ボタンを押すと、往路と復路が切り替わり、点名だけを自動的に逆順で相互反映します。BS・FSなどが入った最終行まで空欄も含めて反転するため、点名の行位置が保たれます。BS・FSなどの観測値はコピーしません。
点名と後視BSの境目にある「＋」で距離列を展開し、「－」で収納します。

`0` は有効な観測値です。FSだけの行も、直前に保持した器械高から高低差と標高を計算します。FSに対応するBSがまだない場合だけ計算できない行として表示します。距離、BS、FS、高低差、標高は小数点以下3桁、往復差はmm単位の整数で表示します。内部値は表示のために過度に丸めません。

「往復差 mm」列は、対応する往路と復路の「既知/仮標高」を逆順で照合し、`ABS（往路標高－復路標高）×1000` を四捨五入した整数で表示します。片方の標高が空欄または計算できていない点は空欄です。復路は往路終点の標高を開始標高として、シートの上から下へ計算します。

上部の「精度区分」で1級、2級、3級、4級を選択します。距離列へ各区間の距離をm単位で入力すると、往路の合計距離（往路未入力時は復路）を片道観測距離 `S（km）` として、往復観測値の較差の許容値を自動計算します。1級は `2.5mm√S`、2級は `5mm√S`、3級は `10mm√S`、4級は `20mm√S` です。距離が未入力の場合は、誤判定を防ぐため「距離待ち」と表示します。基準は国土地理院「作業規程の準則」第65条です。

## 音声入力

1. 入力したいセルをタップまたはクリックします。
2. 画面下部の「🎤 音声モード」ボタンを押し、赤い音声入力モードへ切り替えます。
3. 入力先のセルを選び、赤い「🎤 聞き取る」ボタンを押して日本語で値を話します。
4. 認識値を端末が復唱してカーソルを移動した後も、赤い音声入力モードを維持します。

音声入力モード中は表内のどのセルをタップしてもキーボードを表示せず、タップしたセルを次の音声入力先にします。赤い「🎤 聞き取る」を押した時だけ認識を開始し、聞き取り中は濃い赤色で「■ 聞き取り中（押すと中止）」と表示します。このボタンをもう一度押すと、途中の認識結果を入力せずに中止して、すぐ赤い待機状態へ戻ります。中止後に遅れて届いた認識イベントは無視され、すぐ次の聞き取りを開始できます。入力と復唱が終わった場合も赤い待機状態へ戻ります。「⌨️ キーボードへ戻る」を押すと、その時点で選択中のセルへフォーカスして通常のキーボード入力へ戻ります。
音声入力モード中も、セルの上から縦・横にスワイプして野帳をスクロールできます。タップとスワイプを分けて処理するため、スクロールでは音声入力を開始しません。
認識結果を復唱した後、点名から進むときは直前の1行だけを左から確認し、BSに数値があればBS、BSが空欄でFSに数値があればFSの真下へ移動します。距離列を表示していて、かつ直前の1行の距離に数値がある場合だけ、点名の次に距離で一度止まります。直前の距離が空欄なら距離セルを飛ばします。距離の入力後は同じ規則でBSまたはFSへ移動します。FSへ入力した場合は次の行の点名へ移動します。移動先で画面キーボードは自動表示しません。
音声入力モード中は表内の全セルを読み取り専用にします。別のセルをタップしてもiPhoneのキーボードは開かず、音声の入力先だけが切り替わります。復唱終了または認識エラー後も音声入力モードを維持します。
数値欄は、小数点を含めて5文字認識した時点で入力を即時確定します。「設定」から復唱速度を変更できます。BS・FSは `0.000`～`9.999` の小数3桁だけを音声値として採用し、音声認識の複数候補が異なる有効値になった場合は誤入力を避けるため入力せず、再入力を案内します。復唱は数字を1桁ずつ読みます。

数値欄では、次のように整形します。

- 「マイナス」→ `-`
- 「点」→ `.`
- 全角数字→半角数字
- 空白、読点、句点を削除

例：「マイナス1点235」→ `-1.235`

後視BS、前視FS、距離は、0～9の数字と小数点だけを入力できます。BS・FSは10m未満に制限され、10m以上は入力エラーとして拒否されます。

設定の「よく使う点名」では、実際に入力する点名と音声での読みを登録できます。たとえば `T-1` と `ティノイチ` を登録すると、点名セルで「ティノイチ」と認識した場合に `T-1` が入力されます。登録した読みは文章の一部でも置換され、ひらがな・カタカナを同じ読みとして扱います。`NO`＝`なんばー`、`.`＝`てん`を登録した場合、「ナンバーテンサン」は `NO.3` になります。

空白の点名セルでは、現在のシートの現在行より上だけを調べて増番候補を作ります。1行目は候補なし、2行目は最大1件、3行目は最大2件、4行目以降は最大3件です。直上の点名種類を第1候補にし、第2・第3候補は別種類の直近点名を使います。同じ種類の最大番号を増番するため、`TP3`・`TP4`の次は`TP5`、`SP-2`の次は`SP-3`になります。同じ種類を複数候補へ出さず、数字だけ、ひらがなを含む点名、増番できない点名は候補にしません。候補ボタンで入力した点名も、設定した読みまたは標準読みで復唱してから通常の音声入力と同じ規則で次のセルへ移動します。

主な音声変換例：`ティーピーの12`→`TP12`、`盛替点4`→`TP4`、`ビーエム1`→`BM1`、`ケービーエム2`→`KBM2`、`ナンバー10`→`NO.10`、`BCさん`→`BC.3`、`SPに`→`SP.2`、`のり肩`→`法肩`、`側溝の底`→`側溝底`、`マンホールの上`→`マンホール天端`。

英字測点は大文字へ統一します。曲線関係は `IP.1`、`BP.1`、`BC.1`、`EC.1`、`EP.1`、`SP.1`、`MC.1`、`KA.1`、`KE.1` のようにピリオド付き、BM・KBM・TBM・TPは `BM1`、`KBM1`、`TBM1`、`TP1` の形式です。点名の復唱では画面文字をそのまま読ませず、手動登録した読みを優先し、その後に標準読み辞書を使います。

点名に含まれる半角・全角の英字は、手入力、音声入力、設定した「よく使う点名」、保存済みデータの読込時を含め、すべて大文字へ統一します。

Web Speech APIに未対応のブラウザでは音声入力ボタンだけが無効になり、キーボード入力・計算・保存などは利用できます。音声認識はブラウザや端末の実装により通信を必要とする場合があります。

## 保存と読込

入力変更後に端末内へ自動保存します。「上書き保存」で即時保存できます。ブラウザを閉じても同じブラウザ・同じサイトURLから往路・復路を自動復元します。「消去」では、往路・復路・全シートのいずれかを選び、確認後に消去できます。シート名の横にある太い青色の矢印で、往路・復路ごとにセル編集、音声入力、候補入力、セル削除、行追加・削除、シート消去を元に戻したりやり直したりできます。

保存項目は、精度区分、音声設定、よく使う点名、往路野帳、復路野帳です。

ブラウザのサイトデータを消去した場合や別端末・別ブラウザでは保存内容を引き継げません。必要に応じてCSVも保存してください。

## CSV出力

「CSV出力」を押すと、現在表示している往路または復路シートをダウンロードします。距離列が画面で非表示でもCSVには含まれます。Excelで文字化けしにくいUTF-8 BOM付きです。

## GitHub Pagesへの公開方法

1. このフォルダー一式をGitHubリポジトリへ登録します。
2. GitHubのリポジトリ画面で **Settings → Pages** を開きます。
3. **Build and deployment** のSourceを **Deploy from a branch** にします。
4. `main` ブランチ、`/(root)` を選んで保存します。
5. 表示された `https://ユーザー名.github.io/リポジトリ名/` を開きます。

PWAと音声認識には安全な接続が必要です。GitHub PagesはHTTPSで配信されます。

## 対応ブラウザ

- Chrome / Edge：推奨。音声入力に対応しやすい構成です。
- Safari（iPhone / iPad）：野帳、計算、保存、CSV、PWAを利用できます。音声認識の可否や挙動はOS・ブラウザ版に依存します。
- Firefox：野帳、計算、保存、CSVは利用できます。Web Speech APIが使えない場合はキーボード入力になります。

## 制限事項

- 保存先は端末内のlocalStorageで、現時点では1現場分です。
- 音声認識の精度・可否は端末、ブラウザ、周囲の騒音、ネットワーク状態に依存します。
- オフライン時も画面、計算、保存、CSVは利用できますが、音声認識は利用できない場合があります。
- このアプリは無保証です。重要な成果や判断は必ず正式な機器・手順で確認してください。
